# 获利模式候选库

这是一张启发价值捕获的地图，不是答案库、评分表或支持场景清单，也不声称穷尽所有行业。可靠性来自真实买方证据、成立前提和失败条件，不来自模式数量。开放式问题可以先看`可联系付款方 → 问题确认 → 明确报价 → 有成本承诺 → 到账 / 复购`哪里最值得推进；用户要求全面比较时可以扩展，不受表中模式或候选数量限制。

| 获利逻辑 | 谁付钱 | 真正购买什么 | 收钱事件 | 最小验证 | 关键前提 |
|---|---|---|---|---|---|
| 标准化产品 | 使用者或采购方 | 边界清楚、可重复交付的结果 | 下单 / 付款 | 对目标买方发明确商品与价格 | 商品、价格、交付和售后清楚 |
| 产品化服务 | 有明确问题的个人或组织 | 固定范围、周期和结果 | 订金 / 项目款 | 接受范围与价格并付订金 | 能控制范围、修改和工时 |
| 定制项目 | 问题重要且无法完全标准化的组织 | 特定结果与责任 | 里程碑款 / 项目款 | 决策人确认范围并进入合同或采购 | 需求、验收、变更和责任清楚 |
| 付费诊断 / 审计 | 问题昂贵但路线不清的一方 | 判断、路线、边界或风险降低 | 预约付款 / 诊断费 | 为诊断本身接受价格 | 诊断有独立价值，不是伪装售前 |
| 顾问保留 / 运维支持 | 持续需要响应或判断的组织 | 可用性、响应、复盘和责任 | 周期服务费 / 续约 | 为明确服务级别接受一个周期 | 持续问题真实、服务边界可控 |
| 订阅 / 会员 | 持续获得价值的使用者或组织 | 连续访问、更新、服务或身份 | 周期扣款 / 续费 | 预售一个周期并观察续用理由 | 价值持续发生，不是一次性产品分期 |
| 按量 / 席位计费 | 使用量可记录的一方 | 实际调用、席位或处理量 | 用量账单 / 席位费 | 接受计量单位与首笔付费使用 | 计量可信、边际成本可控 |
| 租赁 / 使用权 | 暂时使用比持有更合理的一方 | 一段时间的资产或能力使用权 | 租金 / 押金 | 接受期限、责任与首期付款 | 权属、损耗、保险和退出清楚 |
| 许可 / 授权 | 需要合法使用资产的组织 | IP、品牌、内容、技术或区域权利 | 许可合同 / 周期费用 | 接受使用范围与许可报价 | 权属清楚、范围可执行 |
| 白标 / OEM / 嵌入式能力 | 已有客户但缺产品能力的组织 | 可贴牌、集成或转售的能力 | 授权费 / 项目费 / 用量费 | 确认目标客户与集成责任后进入报价 | 品牌、质量、支持和责任分配清楚 |
| 免费核心 + 企业服务 | 企业预算方 | 私有部署、治理、集成、培训或责任 | 项目费 / 支持费 | 现有企业询问接受具体付费范围 | 免费核心完整，企业需求真实不同 |
| 数据 / 报告 / API | 需要持续判断的组织 | 合法、独特、及时的洞察 | 报告费 / 订阅 / API费 | 用有权样本让买方接受范围与价格 | 数据权利、质量、隐私和更新成本清楚 |
| 培训 / 认证 | 个人、雇主或生态伙伴 | 能力提升、实施方法或可信证明 | 报名 / 认证费 | 为明确学习结果报名付款 | 结果可检验，认证不误导 |
| 广告 / 赞助 | 想触达特定人群的品牌 | 合格注意力、内容或活动权益 | 发布费 / 赞助费 | 目标赞助方接受受众证据与权益报价 | 受众明确、披露充分、质量可证明 |
| 合格线索 / 佣金 / 推荐 | 获得交易或客户的一方 | 筛选、匹配、介绍或成交贡献 | 线索费 / 成交佣金 / 结算 | 双方书面接受定义、归因与费用 | 双方知情、权限与持续价值成立 |
| 平台抽成 | 供需一侧或双方 | 发现、信任、交易、结算和治理 | 每笔交易 / 服务费 | 完成一笔平台不可替代的真实交易 | 两侧价值成立，不靠阻断直接沟通 |
| 结果付费 / 收益分成 | 能验证增量结果的一方 | 共同承担执行与结果风险 | 达标结算 / 分成 | 书面锁定基线、归因、上限和结算 | 结果可归因、可审计、风险可承受 |
| 预售 / 众筹 | 愿意等待交付的早期买方或支持者 | 未来交付、优先权或共同推进 | 预付款 / 达标放款 | 在交付前获得真实付款 | 交付能力、退款、期限和平台规则清楚 |

## 匹配与排序

1. 先找已有产品、能力、买方、旧客、流量、权利和交付证据，不从零编造。
2. 开放式下一步问题通常先看商业链最早且最有杠杆的断点；若用户问长期战略、组合设计或完整审计，则按问题展开。
3. 比较候选时区分真正不同的价值捕获逻辑，不把价格、套餐或渠道变化冒充新模式。
4. 可参考：`权限可做 → 已有买方证据 → 离收钱近 → 可验证 → 能交付且经济性可能成立`；具体权重服从用户目标和项目现实。
5. 需要商业证据时，优先请求当下可行的最高级：`到账 / 订金 / 订单 / 复购 > 接受价格 / 签署 / PO / 启动采购 > 决策投入 / 真实资料`。
6. 只有第一层是收钱证据；其余只能证明向前移动。播放、点赞、Star、会议和口头兴趣不算。
7. 广告、佣金、数据、结果分成和平台模式必须额外核验质量、归因、披露、权利、单位经济和监管边界。
8. 订阅要求持续价值；平台要求交易或治理价值；免费核心商业化不能靠故意阉割免费版。

## 新获利模式内容卡

发现新的公开模式时，按同一结构记录：

```text
模式名称：
谁在付钱：
真正购买什么：
什么事件产生收入：
成立需要哪些前提：
商业链第一个断点：
当下可请求的最高级证据：
七天最小验证：
最容易失败在哪里：
公开来源与日期：
```

没有当前来源和真实证据时，标记为`假设`或`待核`，不得写成“已验证赚钱模式”。
