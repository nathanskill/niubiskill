# 合成案例 02｜餐饮菜单自由设计师

> `SYNTHETIC / 合成`：本案例由虚构角色和跨行业结构组成，不对应任何真实客户、收入或项目。

## 输入

> 周会为新开业的小餐馆设计菜单，准备提供固定范围、七天交付的服务。他还没有客户，想从一家餐馆开始。本周应该做什么？

# NIUBI Skill

## A. 商业闭环卡

### 一句话判断

这是简单直销，不需要编造平台生态；当前只需要让一位合格餐馆老板用真实菜单和一次沟通证明需求。

### 信息状态

- 全部业务设定：`SYNTHETIC / 合成`
- 周的设计能力和作品：`用户自述（案例设定）`
- 餐馆的预算、时间和修改需求：`待核`
- 固定套餐是否盈利：`假设`

### 一链：最小角色链

`周（设计与交付） ↔ 餐馆老板（使用、决策、付款）`

小红书或地图平台只是入口，不是客户。

### 四流

- **价值流：** 周提供菜单信息结构、视觉设计和可打印文件；餐馆老板提供业务资料、反馈和验收。
- **钱流：** 餐馆老板支付定金和尾款；触发点、退款和修改边界必须写进报价。
- **权力 / 决策与权限流：** 老板决定购买和验收；周控制设计范围、排期和源文件交付条件。
- **信息流：** 老板提供菜品、价格、品牌和打印规格；周用一次确认和有限修改完成交付。

### 用户主位置与价值捕获依据

周是直接服务提供者，依据明确范围、报价、定金和可验收交付捕获价值。

### 持续价值测试

没有需要保护的中间位置。客户持续选择周，应因为作品质量、交付速度、范围清楚和修改治理，而不是因为信息不透明。

### 当前ABCD位置

`A — Acquisition`。当前没有合格买家对话证据。

### 一环：唯一瓶颈关系

`周 → 一位新开业餐馆老板`

## B. 七天承诺实验

### 待证伪假设

如果固定范围套餐与新开业餐馆的真实需求匹配，10家合格对象中至少1家会提供菜单资料并确认15分钟沟通。

### 一诺：本周有成本承诺

老板发送现有菜单资料，并确认一次15分钟需求沟通。支付定金属于更强证据，但不在第一次触达中强迫完成。

### 一入口

一家公开标明“即将开业”且有官方联系方式的餐馆账号。

### 唯一行动与行动资产

```text
你好，我看到你们正在准备开业。我专做固定范围的餐饮菜单设计：1个主菜单、2轮文字修改、7天交付可打印文件。

如果你愿意，我可以先用15分钟确认菜品数量、现有品牌素材和打印规格，再判断这个套餐是否适合；不适合我会直接说明。

你可以先发一份去除私人信息的现有菜单或菜品清单。我不会公开或转发材料。
```

### 七天实验与证据

只联系10家符合公开条件的餐馆，每家使用个性化首句。记录：送达、回复、资料、沟通、报价、定金。只有收到资料并确认沟通才算合格前进。

### 样本/成本上限与停止条件

最多10家、2小时研究与触达，不追加第二渠道。无回复只说明当前对象、入口和动作未产生证据，不证明餐饮设计没有需求。

### Economics / Permission / Risk

- **Economics：** 报价必须覆盖预计工时、软件/字体/打印测试成本和两轮修改；具体数字待周核算。
- **Permission：** 只使用公开商业联系方式；不批量骚扰，不把练习作品冒充客户案例。
- **Risk：** 客户持续要求超范围修改、无法提供合法图片/字体或可接受价格低于成本底线时停止。

### 第7天：继续、修改一个变量或停止

- 出现合格沟通：进入B，提交固定范围报价；
- 送达但无人提交资料：只修改入口或首句中的一个；
- 多轮测试后价格低于成本：停止该套餐。

### 今天完成的一件事

列出第一家符合条件的餐馆，并发送一条个性化私信。
