# 合成案例 03｜开源工具作者与企业付款方

> `SYNTHETIC / 合成`：本案例由虚构角色和跨行业结构组成，不对应任何真实作者、仓库、公司、收入或项目。

## 输入

> 陈维护一个开源文档检索工具，个人开发者免费使用并给了不少Star，但没有收入。几家小型咨询公司询问过私有部署、权限管理和员工培训。陈不想把核心开源版本阉割成收费版，应该如何验证商业化？

# NIUBI Skill

## A. 商业闭环卡

### 一句话判断

免费个人用户不是唯一付款方；陈应保持核心工具可用，把企业的私有部署、权限、培训和持续责任组织成一个有边界的付费试点。

### 信息状态

- 全部业务设定：`SYNTHETIC / 合成`
- Star和企业询问：`用户自述（案例设定）`
- 企业是否有预算、数据权限和真实使用场景：`待核`
- 私有部署是否可被陈稳定交付：`假设`

### 一链：最小角色链

`个人使用者 → 开源项目/陈 ← 企业使用团队 ← 企业决策与预算方`

### 四流

- **价值流：** 开源项目为个人和企业提供基础能力；陈可额外提供部署诊断、权限设计、培训和持续支持。
- **钱流：** 免费使用不产生直接收入；企业预算方可为限定范围的实施和支持付款。价格、成本和付款时点待核。
- **权力 / 决策与权限流：** 陈控制项目维护方向和自己提供的服务范围；企业控制采购、内部数据、环境和上线审批；开源许可证约束双方的使用权。
- **信息流：** 社区问题形成公共改进；企业需求、环境和安全限制需要在授权范围内进入私有实施流程。

### 用户主位置与价值捕获依据

陈既是开源作者，也是企业实施服务提供者。其公共价值来自软件与可信维护记录；其收费依据是企业专属诊断、实施、培训、责任和响应，而不是阻止别人使用核心代码。

### 持续价值测试

企业可以自行使用开源代码，但仍可能为更快部署、权限治理、责任归属和员工采用付费。如果陈只能靠隐藏安装方法收费，商业位置不成立。

### 当前ABCD位置

`B — Bargaining`。已有询问，但没有带预算、范围和负责人投入的真实承诺。

### 一环：唯一瓶颈关系

`陈 → 一家已询问过的咨询公司决策者`

## B. 七天承诺实验

### 待证伪假设

如果企业的真实缺口是实施、权限和采用责任，而不是免费技术咨询，一家已有询问的企业会投入具名负责人、环境清单和采购动作。

### 一诺：本周有成本承诺

企业指定一位负责人，提供一个去敏后的部署环境清单，并接受一次付费或可抵扣的90分钟范围诊断。

### 一入口

回复一家已有私有部署询问的企业邮件，不进行广泛冷启动。

### 唯一行动与行动资产

```text
主题：私有部署范围诊断——先确认是否值得做

核心开源版本继续免费可用。企业服务只覆盖你们专属的部署环境、权限、培训和支持责任。

建议先做一次90分钟范围诊断，产出：
1. 当前环境与数据边界；
2. 必需与可选功能；
3. 双方责任和停止条件；
4. 试点范围、工时与报价依据。

开始条件：指定一位项目负责人；提供去敏后的环境清单；确认资料拥有合法使用权。

如果诊断后发现自行部署更合适，我会直接写明，不推动后续项目。
```

### 七天实验与证据

只联系一家已经询问过的企业。记录是否出现：具名负责人、环境清单、已接受的诊断时间和付款/采购动作。Star、下载量和“以后可能需要”不计入。

### 样本/成本上限与停止条件

只处理1家企业，陈的免费准备上限为3小时。不接收未授权数据，不承诺无限定制，不因一次未回复推断企业市场不存在。

### Economics / Permission / Risk

- **Economics：** 诊断价格至少覆盖准备、会议和书面输出；后续试点另行定价。具体数字待核。
- **Permission：** 检查开源许可证、第三方组件、企业数据权利和保密边界；不要把私有需求直接公开成路线图。
- **Risk：** 陈无法承担支持责任、企业无负责人/预算、数据权限不清或需求本质是无限定制时停止。

### 第7天：继续、修改一个变量或停止

- 出现负责人、环境清单和采购动作：继续设计小试；
- 只有技术兴趣：只修改承诺，不增加功能；
- 企业拒绝范围和责任边界：停止定制。

### 今天完成的一件事

向一家已有询问的企业发送范围诊断说明，并要求具名负责人和去敏环境清单。
