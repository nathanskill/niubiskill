# Tests

`scenarios.json` 是行为验收规范，不是模型跑分结果。每个场景写明真实用户输入、必须出现的行为和不得出现的行为。

`python3 scripts/validate_repo.py` 只检查仓库结构、Skill 约束、示例长度和场景规范是否完整；它不会调用模型，也不代表 17 个场景已经全部通过。

比较版本时，应让同一模型在同一输入、同一审计口径下分别生成结果，再独立检查：

- 是否只有一个赚钱点、一条路线和一份材料；
- 是否明确停止一项不产生商业证据的生产惯性；
- 推荐生产任务时，是否同时存在具体买方、明确成交阻塞、最小修复和修完后的承诺动作；
- 有旧客、询盘或可联系付款方时，是否优先成交而不是继续做内容或功能；
- 获利模式是否来自不同价值捕获逻辑，并选择了证据最强、离收钱最近的一种；
- 是否请求当下可行的最高级证据，并区分收钱证据与前进证据；
- 关键商业数字是证据值、推导值、实验参数还是未知；
- 假设是否被写成事实；
- 对外材料是否编造价格、能力、案例、稀缺或客户反馈；
- 闸门是否只约束当前动作；
- 输出长度与材料可用性。

没有原始输出、统一口径和可复算记录时，不发布胜率或改进百分比。
