# Development runs

本目录保存候选版开发期原始输出。它们用于复核失败和设计取舍，不是模型榜单，也不证明真实用户、执行率、成交或收入。

## 目录说明

- `2026-07-20-insight-first/`：公开 `v0.4.0` 与早期强约束候选的冻结对照；该候选后因模板化和C06触达诊断失败被放弃。
- `2026-07-20-insight-first-r2/`：第一次“高自由度”改写，验证比较、完整审计、访谈和发散可以服从用户；随后仍发现事实纪律问题。
- `2026-07-20-insight-first-r3/`：12场景完整高自由度测试。独立盲审为8通过、4失败，红队另发现6项P1；明确不是 `PROVISIONAL PASS`。
- `r4`至`r7`：针对失败点的开发期复测，不是完整验收。对应Skill SHA-256依次为：
  - `r4`: `b9d65af9864441a33fa6c7d4d555ebbf55298f766798c56850d56ed773240479`
  - `r5`: `28ed73cd61daeec1b6fb5a4e4548cdfe81f3c716f83f0a3fa4a666dd55ec8019`
  - `r6`: `d1afd5fcfc6e25cb718445bb887c82088bde17b155f37fb250eb8a3715901bca`

`r7`只复测极简输入C01，仍出现未经用户确认的具体交付能力，因此最终工作区候选仍不得发布。

所有场景都是回归探针，不是Skill的能力边界或唯一正确答案。后续只有在同一最终SHA上重跑完整12场景、独立审计无发布阻断项后，才可讨论候选发布。
