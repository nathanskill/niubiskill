# NIUBI 赚钱纠偏

**结论：** 暂时不要扩大获客。继续把现有商品卖给现有付款方，只有“订单收入扣除退款、人工交付及其他订单变动成本后仍为正”才算有效变现；当前单位经济性为**待核**。  
**现在：** 先停新增投放、渠道扩张和促销；只走**暂停**——项目已有收入（用户自述），但退款和人工成本上升（用户自述），继续放量可能放大亏损。  
**本周：** 在订单系统对最近30天的已付款客户逐单核算真实净贡献，并按退款原因分类。  
**7天只看：** 最近30天已付款订单的合计净贡献是否为正（有效核算：每笔订单均有收入、退款、人工工时成本及其他变动成本记录；保存：订单级核算表和退款原因记录）。为正→仅恢复小规模获客；不为正→只改**交付范围与验收边界**；无法可靠核算或确认无法履约→继续停获客。

**数字依据：**

- “最近30天”：实验参数，用于观察当前成本变化。
- 净贡献＝实收收入－退款－人工交付成本－其他订单变动成本。
- 获客成本、退款率、单笔人工工时及净贡献：未知，需从真实记录核算。

```text
交付中最耗人工的环节：
退款是否由范围、预期、质量、时效或其他原因引起：
```

第七天汇总：

```text
订单总实收：【占位符】
退款总额：【占位符】
人工交付总成本：【占位符】
其他变动成本：【占位符】
合计净贡献：【占位符】

决定：
合计净贡献为正，且主要退款原因已有可执行修复 → 小规模恢复获客。
合计净贡献不为正 → 继续停获客，只收紧一次交付范围与验收边界后复测。

```

发出前先替换全部【占位符】，并删除任何你不能背书的句子。
