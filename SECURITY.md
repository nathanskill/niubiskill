# Security Policy

## 支持版本

当前只维护最新发布版本。

## 私下报告

如果发现以下问题，请不要创建包含敏感细节的公开 Issue：

- 密钥、令牌或私密仓库信息；
- 真实客户、联系人、雇主或第三方机密；
- 可以反向识别自然人的案例材料；
- 其他可能造成现实伤害的安全问题。

请使用 GitHub 仓库的 **Security → Report a vulnerability** 私密报告入口。若该入口尚未启用，只创建一条不含敏感内容的 Issue，请求维护者开启私密沟通渠道。

不要在报告中提交与复现无关的个人数据。

公开QQ和微信仅用于一般合作与反馈，不是安全材料传输渠道。不要通过即时通讯发送密钥、身份证件、账户、客户名单或其他敏感资料。
