# NIUBI Skill

## 你不是缺生产力，你是忘了赚钱。

项目做着做着，最容易沉浸在功能、内容和AI生产力里，把收费验证推迟到“再完善一点”。

**NIUBI Skill是项目的赚钱纠偏器。**

在你继续开发、继续发内容或继续投入之前，它先判断你正在做的事情是否真的接近收入，再找出一个离真实收钱最近的赚钱点。

> **少做一点，先拿到一次真实商业反馈。**

```bash
npx -y skills add nathanskill/niubiskill -g --all
```

## 它只做两件事

### 1. 提醒你：该赚钱了

做产品容易获得完成感，报价、收费和面对拒绝更难，所以人会本能地继续生产。

NIUBI Skill会叫停一项不产生商业证据的开发、内容或投入。只有真实买方明确提出阻塞，而且修复后可以立即推进承诺或付款时，才允许继续生产。

### 2. 给出最可行的获利路径

后台从不同价值捕获逻辑中最多比较3个，只推荐1个：

```text
谁付钱 → 为什么付 → 买什么 → 什么事件产生收入
```

它不把模式清单扔给你，而是按照：

```text
权限可做 → 已有买方证据 → 离收钱近 → 七天可验证 → 能交付且经济性可能成立
```

选出一个本周动作、一份可以直接使用的材料和一个七天证据。

[查看获利模式候选库](skills/niubiskill/references/monetization-patterns.md)

## 什么时候必须跑一次

- 准备继续开发新功能、再做一个AI工具；
- 准备重做网站、页面、品牌或内容计划；
- 准备招人、自动化或追加投入；
- 有用户、粉丝、Star或流量，却没有收入；
- 已经有人咨询，却没有报价、承诺或付款；
- 团队一直很忙，但说不清谁会为什么付钱。

## 3步，找到离钱最近的赚钱点

### 1. 安装

```bash
npx -y skills add nathanskill/niubiskill -g --all
```

安装或更新后重新打开Agent。

### 2. 把项目丢进去

```text
$niubiskill https://你的项目网址
告诉我本周先停什么，再找一个离真实收钱最近的赚钱点。
```

没有网址，一句话也行：

```text
$niubiskill
我在做【项目】，现在有【用户、流量、询问或收入】，下一步原本准备【继续做什么】。帮我纠偏并找赚钱点。
```

你只需要描述项目。获利模式、ABCD、证据强弱、权限边界和候选比较都由Skill在后台完成。

### 3. 当天行动

```text
结论：谁为什么付钱、买什么、什么事件才算收钱
现在：本周先停什么，只走引流 / 成交 / 暂停
本周：一个入口、一个目标角色、一个动作
7天：一个收钱或前进事件
直接用：一份完整材料
```

除行动材料外，默认答案应能在一屏读完。

## 什么才算商业反馈

```text
到账 / 订金 / 订单 / 复购
> 接受价格 / 签署 / PO / 启动采购
> 决策人投入 / 提交真实资料
> 播放 / 点赞 / Star / 会议 / 客套兴趣
```

只有第一层是收钱证据，中间两层只是前进证据，最后一层不算商业结果。默认请求当下可行的最高一级。

## 怎么判断先做什么

只推进商业链的第一个断点：

```text
可联系付款方 → 问题确认 → 明确报价 → 有成本承诺 → 到账 / 复购
```

- 没有可联系的合格付款方：引流；
- 已有目标对象、用户、旧客或询问：成交；
- 当前动作的关键权限不清：暂停并取得决定。

内容只是可能的获客入口，不是默认答案；发布内容本身也不是证据。

## 原则

NIUBI Skill不保证赚钱，也不编造价格、能力、案例或客户反馈。一次付款不证明市场成立；一次失败也不能判死项目。涉及数据、IP、雇佣、平台或受监管活动时，只允许有权限、能履约的下一步。
