# NIUBI Skill

**蒸馏全行业TOP1，打开0到1000w赚钱之路的牛逼skill。**

## 获客 + 成交 = 赚钱

只要你有一项能力、一段经验、一个产品、一种资源或一个想法，都可以从这里开始。

NIUBI Skill 把全行业反复有效的赚钱动作，压缩成 Nathan 的 **ABCD**：A 获客、B 成交、C 维护、D 开发。

你告诉它你会什么、有什么、已经做到哪一步；它帮你确定卖什么、卖给谁、去哪里找到客户，以及本周怎样推进一次真实成交。

可在 Codex、Claude Code、Cursor 等支持 Skill / system prompt 的 Agent 中使用。

## 30 秒开始

Codex 一行安装（需要 Node.js 18+）：

```bash
npx skills add nathanskill/niubiskill --skill niubiskill -g -a codex -y
```

或者手动克隆：

```bash
git clone https://github.com/nathanskill/niubiskill.git
cd niubiskill
```

将 `skills/niubiskill` 复制到 Agent 的 Skill 目录。Codex 默认目录示例：

```bash
mkdir -p ~/.codex/skills
cp -R skills/niubiskill ~/.codex/skills/niubiskill
```

重新打开 Agent 后直接调用：

```text
$niubiskill

我会/拥有[能力、经验、产品、资源或想法]，目前已经做到[当前进度]，想实现[具体结果]。
请用 ABCD 帮我确定卖什么、卖给谁、从哪里获客，以及本周怎样推进一次真实成交。
```

## 一个合成案例：会做设计，怎样拿到第一笔订单

> 以下为跨行业合成案例，只用于展示方法，不代表任何真实客户、项目或结果。

周会为小餐馆设计菜单，但还没有客户。他准备推出固定范围、七天交付的套餐，先验证是否有人愿意拿出真实资料并讨论购买。

NIUBI Skill 会把这个局面压缩为：

- **卖什么：** 一个主菜单、两轮文字修改、七天交付可打印文件。
- **卖给谁：** 正在筹备开业、确实需要菜单的餐馆老板。
- **当前 ABCD：** `A — Acquisition`。现在缺的不是继续学设计，而是一位愿意讨论真实需求的人。
- **一入口：** 一家公开标明“即将开业”并留有官方联系方式的餐馆。
- **一个行动资产：** 一条针对真实开业信息写的私信。
- **七天证据：** 对方提供菜单资料并确认一次需求沟通；点赞和客套回复不算成交进展。

[查看完整合成案例](examples/02-menu-designer.md)

## 它具体怎么帮你变现

每次先回答四个问题：你卖什么、卖给谁、怎么找到他、这周怎样推进一次真实成交。

如果只是简单直销，就直接做获客和成交；只有当使用者、付款者、决策者不是同一个人时，才展开角色、钱、权力和信息分析。

最后只给两个结果：

```text
A. 商业闭环卡
   一链：最多五个关键角色
   四流：价值 / 钱 / 权力与权限 / 信息
   一位：用户的主位置与价值捕获依据
   一环：当前唯一瓶颈关系

B. 七天承诺实验
   一诺：一个有成本承诺
   一入口：一个有权使用的接触点
   一资产：一份可以立即使用的成品
   一周：一次执行与结果回填
```

最终只做三种决定：继续、修改一个变量、停止。

## 什么时候使用

- 会做一件事，却不知道怎样把它变成可卖的产品；
- 有产品或服务，却不知道第一批客户在哪里；
- 有人询问或表示兴趣，却迟迟没有真实成交；
- 已经有收入，但获客、成交、交付、复购或转介绍无法复制；
- 有内容、关系、渠道、社群或其他资源，却没有稳定的价值与收费方式；
- 想从第一次付费验证继续走向可复制增长，却不知道当前唯一瓶颈。

## 它和普通销售提示词有什么不同

普通销售提示词通常默认“一个卖方 + 一个买方”。NIUBI Skill 先检查：

1. 受益者、使用者、决策者、付款者、分配者是否为同一主体；
2. 用户创造了什么价值，凭什么获得工资、奖金、服务费、分成、订阅、股权或其他回报；
3. 谁控制价格、客户、数据、结算、暂停和退出；
4. 即使参与者可以直接沟通，为什么仍愿意让用户参与；
5. 七天内哪一个承诺最能证明闭环真的前进。

## ABCD

- **A — Acquisition / 获客：** 找到真正可能购买、合作或批准的人。
- **B — Bargaining / 成交：** 把“有兴趣”推进为明确规则、下一步和真实投入。
- **C — Customer Care / 维护：** 把客户服务好，让交付、回款、续费、转介绍和信任持续运转。
- **D — Development / 开发：** 持续开发产品、能力、案例、内容、工具和合作伙伴，为下一轮获客与成交增加弹药。

它们不是一次性漏斗，而是循环：`D → A → B → C → D`。

## 边界

NIUBI Skill 不帮助：

- 保证赚钱、成交、收益或投资回报；
- 隐藏佣金、重大利益冲突或真实商业身份；
- 盗版、破解、刷量、假账号、骚扰或规避平台和监管；
- 未经同意分享联系人、客户材料、公司数据或第三方知识产权；
- 通过阻止合作方直接沟通来人为制造依赖。

遇到雇佣、跨境、数据、平台或受监管活动，Skill 会先做权限闸门。规则不清时只输出内部结构和待核清单，不生成外部招揽方案。
