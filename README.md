# NIUBI Skill

## 帮您走好赚钱路。

**谁付钱。为什么付。卖什么。先引流？如何快速成交。**

## AI把做项目的门槛降到了最低，却没有把赚钱变简单

以前需要团队、资金和技术才能做出的产品，现在一个人借助AI就能完成。当“把东西做出来”不再稀缺，真正稀缺的，是让对的人看见它、相信它，并愿意为它付钱。

> **引流「产出侧」 + 成交「销售侧」 = 变现真相。**
>
> **获客 + 成交 = 赚钱！！**

- **产出侧：** 把价值变成产品、内容、案例、工具和渠道，让对的人看见你。
- **销售侧：** 把兴趣推进为信任、报价、承诺、订单和复购，让愿意的人付钱。

从价值产出到真实收入的完整链路，才是AI时代最难被替代的能力。

## 不是让你想得更多，而是让项目离钱更近

3个赚钱点选1个落地。本周，只做最赚钱的那一步。

## 3步，找到离钱最近的赚钱点

无需学 ABCD 全量框架，不用填长表，也不用写商业计划。

### 1. 一行安装

适用于 Codex、Claude Code 等支持 Agent Skill 的工具（需要 Node.js 18+）：

```bash
npx -y skills add nathanskill/niubiskill -g --all
```

安装完重新打开 Agent。更新时，重新运行同一条命令。

### 2. 把项目丢进去

Codex 中直接输入：

```text
$niubiskill https://你的项目网址
帮我找赚钱点。
```

没有网址，一句话也行：

```text
$niubiskill
我在做【项目、产品或服务】，现在已经有【用户、流量、询问或收入】，但卡在【问题】。帮我找赚钱点。
```

直接说：“用 NIUBI Skill 帮我找这个项目的赚钱点。”

### 3. 拿到秘籍，当天就做

NIUBI Skill 不把一堆选项丢回给你。它会直接给出：

- 赚钱点；
- 唯一推荐；
- 当前只做引流、成交，还是先暂停；
- 1 份可以直接发的内容或材料；
- 1 次七天验证。

**你只需要描述项目。赚钱模式、ABCD 和风险判断，由 Skill 在后台完成。**

## 一次给一张赚钱点卡秘籍

```text

最多3个候选赚钱点

当前路线：引流 / 成交 / 暂停

```

## 两条路线！

| | 引流 | 成交 |
|---|---|---|
| 要解决的问题 | 怎么让对的人看见你 | 怎么让有意向的人付钱 |
| 主要方式 | 内容、案例、搜索、免费工具、合作、推荐 | 诊断、报价、提案、试点、跟进、复购 |
| 行动资产 | 一篇内容、一张案例、一页工具或一个落地页 | 一条私信、一份报价、一页提案或一封跟进 |
| 有效证据 | 合格咨询、提交真实资料、预约或留资 | 接受价格、投入时间、试点、订金、订单或合同 |
| 无效证据 | 播放量、点赞、泛关注 | “有兴趣”“再看看”“保持联系” |

没有合格线索，先引流。已经有人来但没人付钱，先成交。权限不清，先暂停。

## 什么才算赚钱点？

赚钱点必须说清六件事：

1. 你现在有什么；
2. 谁会付钱；
3. 对方为什么付钱；
4. 你到底卖什么；
5. 什么事件触发收入；
6. 七天内怎样验证。

说不清付款方、商品和收钱事件的，只是想法，不是赚钱点。

## 每个新商业模式，都是一条新内容

市场上每出现一种经过验证的赚钱方式，就拆成一张内容卡，再变成 NIUBI Skill 的新弹药：

```text
谁付钱
真正购买什么
什么事件产生收入
主要靠引流还是成交
成立需要什么前提
哪些项目可以迁移
七天怎样验证
最容易失败在哪里
```

当前内置候选包括一次性产品、产品化服务、付费诊断、订阅、按量计费、许可、免费核心加企业服务、佣金、平台抽成、广告赞助、培训认证和数据报告。

[查看赚钱模式候选库](skills/niubiskill/references/monetization-patterns.md)

## ABCD是内核，不是负担

- **A — Acquisition：** 有没有对的人进来；
- **B — Bargaining：** 有没有把兴趣变成承诺；
- **C — Customer Care：** 有没有复购、续费、转介绍和扩单。
- **D — Development：** 有没有持续可卖的产品、案例和能力。

用户只看到当前最该做的一步。复杂角色、四流和权限判断只在会改变结论时启动。
