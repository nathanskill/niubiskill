# Contributing

感谢你帮助改进 NIUBI Skill。贡献应让方法更清楚、更可验证或更安全，而不是让仓库变得更大。

## 可以提交

- 可复现的推理错误或结构问题；
- 明确标注的跨行业合成场景；
- 有公开来源和日期的新商业模式内容卡；
- 对赚钱点定义、引流/成交路由、固定输出、证据状态或边界的改进；
- 拼写、链接、安装和兼容性修复；
- 不依赖私人数据的测试标准。

## 不要提交

- 真实姓名、电话、邮箱、社交账号或可识别联系人；
- 聊天记录、CRM、合同、账户、交易、客户名单或雇主内部材料；
- 未经许可的第三方代码、课程、数据库、文章或提示词；
- 收益保证、误导宣传、隐藏返佣、刷量、骚扰或规避方法；
- 仅以“我认识很多人”为价值、但没有权限和分配机制的资源倒卖方案。

真实经历只能先抽象成其他行业的合成结构，并删除所有可能反向识别主体的信息。不要仅替换姓名后提交。

## 证据标记

对重要陈述使用：

- `已核验`
- `用户自述`
- `待核`
- `假设`
- `SYNTHETIC / 合成`

## Pull Request 最小要求

1. 说明解决了什么问题；
2. 说明为什么需要修改，而不是增加更多文件；
3. 添加或更新至少一个验收标准；
4. 运行 `python3 scripts/validate_repo.py`；
5. 声明拥有提交内容的使用权，且不包含机密或个人资料。

新商业模式必须说明谁付钱、购买什么、什么事件产生收入、主要靠引流还是成交、成立前提、可迁移项目、七天验证、失败条件、公开来源和日期。没有来源或结果证据时标记为`假设`或`待核`。
